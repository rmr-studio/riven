Purpose: Interface-first — contract before implementation. Downstream plans in this phase and Phase 3/4 all depend on these types. No runtime behavior; pure types + interface.
Output: ~10 new Kotlin files + one contract-test stub exercising the interface via a fake adapter.

    IngestionAdapter.kt:
    ```kotlin
    package riven.core.service.ingestion.adapter

    import riven.core.models.ingestion.adapter.RecordBatch
    import riven.core.models.ingestion.adapter.SchemaIntrospectionResult
    import riven.core.models.ingestion.adapter.SyncMode

    /**
     * Polyglot ingestion adapter contract. One @Component implementation per [SourceType].
     * Stateless — connection + credentials travel via [AdapterCallContext] per call.
     *
     * Implementations raise [AdapterException] subtypes; Temporal uses the sealed hierarchy
     * for retry classification (Phase 4 wiring).
     */
    interface IngestionAdapter {
        fun syncMode(): SyncMode
        fun introspectSchema(context: AdapterCallContext): SchemaIntrospectionResult
        fun fetchRecords(context: AdapterCallContext, cursor: String?, limit: Int): RecordBatch
    }
    ```

    AdapterCallContext.kt (sealed hierarchy — Phase 1 ships base + Nango variant only; Phase 3 adds Postgres):
    ```kotlin
    package riven.core.service.ingestion.adapter

    import java.time.Instant

    /** Per-call context carrying connection + credentials. Sealed — extended in Phase 3 with PostgresCallContext. */
    sealed class AdapterCallContext {
        abstract val workspaceId: String
    }

    data class NangoCallContext(
        override val workspaceId: String = "",  // TODO Phase 4: source from orchestrator — Phase 1 stub default
        val providerConfigKey: String,
        val connectionId: String,
        val model: String,
        val modifiedAfter: Instant? = null,
    ) : AdapterCallContext()
    ```

    Decision note: `workspaceId` default to `""` acceptable for Phase 1 since NangoAdapter is registered but not runtime-wired. Phase 4 orchestrator passes real workspaceId. Add a `TODO Phase 4` KDoc comment.

    SourceTypeAdapter.kt (qualifier annotation per research Open Q #1 recommendation — ship it now so Phase 4 has nothing to invent):
    ```kotlin
    package riven.core.service.ingestion.adapter

    import org.springframework.beans.factory.annotation.Qualifier
    import riven.core.enums.integration.SourceType

    /**
     * Marks an [IngestionAdapter] @Component as the implementation for a given [SourceType].
     * Phase 4's IngestionOrchestrator consumes a Map<SourceType, IngestionAdapter> assembled from these.
     */
    @Qualifier
    @Target(AnnotationTarget.CLASS)
    @Retention(AnnotationRetention.RUNTIME)
    annotation class SourceTypeAdapter(val value: SourceType)
    ```

    DO NOT build the `@Configuration` factory that assembles the `Map<SourceType, IngestionAdapter>` — that belongs in Plan 03 where NangoAdapter is the first entry and the registry wiring test needs it. Keeping annotation-only here preserves single responsibility.

    ```kotlin
    // AdapterException.kt
    package riven.core.service.ingestion.adapter.exception

    /** Base for all adapter-raised exceptions. Sealed — Temporal retry classification dispatches on leaf types. */
    sealed class AdapterException(message: String, cause: Throwable? = null) : RuntimeException(message, cause)
    ```

    ```kotlin
    // TransientAdapterException.kt — retryable (network blips, rate limits, temporary unavailability)
    package riven.core.service.ingestion.adapter.exception

    class TransientAdapterException(message: String, cause: Throwable? = null) : AdapterException(message, cause)
    ```

    ```kotlin
    // FatalAdapterException.kt — sealed base for non-retryable
    package riven.core.service.ingestion.adapter.exception

    sealed class FatalAdapterException(message: String, cause: Throwable? = null) : AdapterException(message, cause)
    ```

    Five concrete subtypes of FatalAdapterException (each a separate file for import clarity):
    - `AdapterAuthException` — 401/403 auth failure
    - `AdapterConnectionRefusedException` — host unreachable, 404 equivalents, connection refused
    - `AdapterSchemaDriftException` — table/column disappeared between syncs (consumed by Phase 6 HLTH-04)
    - `AdapterUnavailableException` — persistent 5xx or unknown fatal
    - `AdapterCapabilityNotSupportedException` — capability not implemented for this adapter (e.g. Nango introspectSchema)

    Each: `class Foo(message: String, cause: Throwable? = null) : FatalAdapterException(message, cause)`.

    Add a KDoc line at the top of AdapterException.kt noting that Temporal `RetryOptions.Builder.setDoNotRetry(*FatalAdapterException::class.java.sealedSubclasses.map { it.qualifiedName }.toTypedArray())` (or similar) will consume the sealed tree in Phase 4 — do NOT wire Temporal here.

    DO NOT use `NotImplementedError` anywhere. Typed exceptions only.
