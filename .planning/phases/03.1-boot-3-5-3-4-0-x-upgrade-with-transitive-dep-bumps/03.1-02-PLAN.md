Security 7 migration reference surface (from Spring Security 7 migration guide):

Removed / deprecated in Security 7:
- `authorizeRequests { }` → use `authorizeHttpRequests { }` (already preferred in 6.x)
- `antMatchers(...)` → use `requestMatchers(...)`
- Positional DSL like `http.csrf().disable().authorizeRequests()...` — must use lambda `http { csrf { disable() } ... }` form
- `WebSecurityConfigurerAdapter` — already removed pre-7; confirm not used
- `.and()` chain combinator — replaced by lambda nesting

Current (Security 7) canonical form for JWT resource server:
```kotlin
@Bean
fun securityFilterChain(http: HttpSecurity): SecurityFilterChain {
  http {
    csrf { disable() }
    authorizeHttpRequests {
      authorize("/api/v1/public/**", permitAll)
      authorize(anyRequest, authenticated)
    }
    oauth2ResourceServer {
      jwt { jwtDecoder = customJwtDecoder }
    }
    sessionManagement { sessionCreationPolicy = SessionCreationPolicy.STATELESS }
  }
  return http.build()
}
```
