Workflow:

1. Boot the app against each active profile briefly and capture the migrator output. Spring will log every flagged key at bean initialisation — one line per key per profile. Alternative: `./gradlew bootRun` with each profile; but test harness is faster and surfaces test-profile keys too.

   - `spring.mvc.*` reorganisations
   - `management.endpoint.*` reshuffles

   - `logging.pattern.*`
   - `server.*` reorganisations
   Apply across ALL four yml files (application.yml, application-dev.yml, application-integration.yml, application-test.yml). Integration profile may override different keys than dev — both must be clean.

3. Re-run the migrator until the "has been renamed" log is empty. ANY remaining flagged key blocks closure.
