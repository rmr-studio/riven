# Workspace Notes Hub

## Overview

Promote the Notes feature from entity-scoped drawers to a workspace-level browsable, searchable, sortable notes hub. Users can see ALL notes across their workspace in one place, search across content, create notes from anywhere with an entity picker, and view them in a full-screen editor with entity context breadcrumbs.

## Scope

### Core (Approach C)
- **Sidebar panel:** New "Notes" entry in the icon rail with note count badge. Panel shows date-grouped note list with search bar and "New Note" button.
- **Workspace notes list route:** `/dashboard/workspace/[workspaceId]/notes` — sortable/filterable DataTable with columns: Note, Created by, Record (entity), Created at. Date-grouped sections.
- **Full-page note editor route:** `/dashboard/workspace/[workspaceId]/notes/[noteId]` — BlockEditor in full-screen with entity breadcrumb (entity type > entity name), auto-save, delete.
- **Mobile navigation:** Notes entry in mobile nav menu.

### Backend Requirements
- **New endpoint:** `GET /api/v1/notes/workspace/{workspaceId}` — list all notes for workspace with cursor pagination, optional `?search=` full-text search, sorting
- **New endpoint:** `GET /api/v1/notes/workspace/{workspaceId}/{noteId}` — fetch single note by ID
- **Enriched Note model:** Add `entityDisplayName`, `entityTypeKey`, `entityTypeIcon` to Note response (JOIN from entities + entity_types tables)
- **New repo methods:** `findByWorkspaceIdOrderByCreatedAtDesc`, `searchByWorkspaceId`
- **New composite index:** `idx_notes_workspace_created_at ON notes(workspace_id, created_at DESC)` for cursor pagination

### Accepted Expansions
1. Search bar in sidebar panel (workspace-wide full-text search)
2. "New Note" button with entity picker modal
3. Note count badge on Notes sidebar icon
4. Date-grouped timeline in notes list
5. Entity breadcrumb in full-page editor
6. Sort/filter controls (table view) in workspace notes list
7. Mobile navigation access
8. Single note GET endpoint

### Cache Invalidation Strategy
```
  Save/Delete note ──▶ invalidate:
    ['notes', workspaceId, entityId]    (entity drawer)
    ['workspace-notes', workspaceId]    (notes hub)
    ['entities', workspaceId]           (note_count on entity)
```

### Key Decisions
- **Pagination:** Cursor-based (createdAt timestamp), matches NotificationService pattern
- **Entity context:** Enriched in backend Note model (JOIN), not fetched separately
- **Two editing surfaces:** NoteDrawer (entity-scoped) and full-page route share same API + mutation hooks

- **Desktop (≥lg):** Full sidebar panel (25rem), DataTable with all columns, editor max-w-3xl centered
- **Tablet (md):** No sidebar panel, table hides "Created by" column, editor full-width
- **Mobile (<sm):** Card layout replaces table, entity picker uses Sheet (bottom), Notes in mobile hamburger menu

## Engineering Review Decisions (2026-03-20)

1. **Backend-first sequencing:** Backend endpoints + model enrichment → `npm run types` → frontend work. No UI-first development.
2. **Cache invalidation:** Update existing `useSaveNoteMutation` and `useDeleteNoteMutation` to invalidate BOTH entity-scoped (`['notes', wId, entityId]`) AND workspace-scoped (`['workspace-notes', wId]`) query keys. One set of mutation hooks, not duplicated.
3. **Feature module consolidation:** Move all existing entity-scoped note code (`note.service.ts`, `use-notes.ts`, `use-save-note-mutation.ts`, `use-delete-note-mutation.ts`, `note-drawer.tsx`) from `components/feature-modules/entity/` into `components/feature-modules/notes/`. Single source of truth for all note logic.
4. **Date-grouped rows:** Visual-only group headers inserted between flat DataTable rows. No modification to shared DataTable component. Bespoke pattern that serves as reference for future entity cluster grouping.
5. **Sidebar search:** Server-side search via `?search=` param, debounced 300ms. Not client-side filtering.
6. **Note count badge:** Backend includes `totalCount` field in paginated workspace notes response. Sidebar reads count from same query — zero extra API calls.
7. **Breadcrumb navigation:** Follow existing breadcrumb pattern from `entity-types-dashboard.tsx`.
8. **Tests:** All 8 new codepaths must have tests (T1-T8 per test diagram in eng review).
