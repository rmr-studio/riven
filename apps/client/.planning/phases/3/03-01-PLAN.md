Rewrite the relationship form data layer: Zod validation schema, form hook with useFieldArray for target rules, submit handler that maps form values to SaveRelationshipDefinitionRequest, and modal reset logic.

<task type="auto">
  <name>Task 2: Rewrite relationship form hook with new Zod schema, useFieldArray, and submit mapping</name>
  <files>
    components/feature-modules/entity/hooks/form/type/use-relationship-form.ts
  </files>
  <action>
    **Complete rewrite** of `use-relationship-form.ts`. Delete all existing content and replace with the new implementation.

    **A. New Zod schema (`relationshipFormSchema`):**

    Define a `targetRuleSchema` using `z.object()` with:
    - `id: z.string().optional()` -- server-assigned ID for existing rules
    - `ruleType: z.enum(['entity-type', 'semantic-group'])` -- discriminator for UI rendering (NOT sent to API)
    - `targetEntityTypeKey: z.string().optional()` -- store human-readable key, resolve to UUID on submit
    - `semanticTypeConstraint: z.nativeEnum(SemanticGroup).optional()` -- semantic group enum
    - `cardinalityOverride: z.nativeEnum(EntityRelationshipCardinality).optional()` -- override per rule
    - `inverseVisible: z.boolean().default(true)` -- show on target entity
    - `inverseName: z.string().optional()` -- name on target side

    Add `.refine()` for XOR validation: exactly one of `targetEntityTypeKey` or `semanticTypeConstraint` must be set (not both, not neither). Check based on `ruleType`:
    - If `ruleType === 'entity-type'`, `targetEntityTypeKey` must be a non-empty string
    - If `ruleType === 'semantic-group'`, `semanticTypeConstraint` must be defined
    Error message: `'Each rule must have either a target entity type or a semantic group constraint'`
    Path: `['targetEntityTypeKey']`

    Define `relationshipFormSchema` with:
    - `name: z.string().min(1, 'Name is required')`
    - `icon: z.object({ type: z.nativeEnum(IconType).default(IconType.Link2), colour: z.nativeEnum(IconColour).default(IconColour.Neutral) })`
    - `semanticDefinition: z.string().optional()` -- natural language description of the relationship
    - `sourceLimit: z.enum(['ONE', 'UNLIMITED']).default('ONE')`
    - `targetLimit: z.enum(['ONE', 'UNLIMITED']).default('UNLIMITED')`
    - `allowPolymorphic: z.boolean().default(false)`
    - `targetRules: z.array(targetRuleSchema).default([])`

    Export the schema and inferred type: `export type RelationshipFormValues = z.infer<typeof relationshipFormSchema>;`

    **B. Default values constant:**

    Create a `DEFAULT_RELATIONSHIP_FORM_VALUES` constant matching `RelationshipFormValues` with sensible defaults:
    - `name: ''`
    - `icon: { type: IconType.Link2, colour: IconColour.Neutral }`
    - `semanticDefinition: ''`
    - `sourceLimit: 'ONE'`
    - `targetLimit: 'UNLIMITED'`
    - `allowPolymorphic: false`
    - `targetRules: []`

    **C. Form hook (`useRelationshipForm`):**

    New function signature (rename from `useEntityTypeRelationshipForm` to `useRelationshipForm` for clarity):
    ```typescript
    export function useRelationshipForm(
      workspaceId: string,
      entityType: EntityType,
      availableTypes: EntityType[],
      open: boolean,
      onSave: () => void,
      onCancel: () => void,
      relationship?: RelationshipDefinition,
    ): UseRelationshipFormReturn
    ```

    Return type:
    ```typescript
    export interface UseRelationshipFormReturn {
      form: UseFormReturn<RelationshipFormValues>;
      handleSubmit: (values: RelationshipFormValues) => void;
      handleReset: () => void;
      mode: 'create' | 'edit';
      targetRuleFieldArray: UseFieldArrayReturn<RelationshipFormValues, 'targetRules'>;
    }
    ```

    Implementation details:

    1. **useForm** with `zodResolver(relationshipFormSchema)` and `defaultValues: DEFAULT_RELATIONSHIP_FORM_VALUES`

    2. **useFieldArray** with `control: form.control, name: 'targetRules'`. Return the field array in the hook return so UI components can call `append`, `remove`, `fields` directly. Do NOT set `keyName` -- RHF v7+ handles `id` field coexistence correctly.

    3. **Edit mode population (useEffect):** When `open && relationship`, call `form.reset({...})` mapping from `RelationshipDefinition` read model to form values:
       - `name: relationship.name`
       - `icon: relationship.icon` (Icon object maps directly)
       - `semanticDefinition: ''` (semantic metadata not on response -- leave empty for now, see Open Question 1 in research)
       - `sourceLimit` / `targetLimit`: use `processCardinalityToLimits(relationship.cardinalityDefault)` then map `RelationshipLimit.SINGULAR -> 'ONE'`, `RelationshipLimit.MANY -> 'UNLIMITED'`
       - `allowPolymorphic: relationship.allowPolymorphic`
       - `targetRules`: map `relationship.targetRules` to form shape, reverse-resolving UUID to key: `availableTypes.find(et => et.id === rule.targetEntityTypeId)?.key`. Set `ruleType` based on which field is present.

    4. **Modal close reset (useEffect):** When `!open`, use `setTimeout(() => form.reset(DEFAULT_RELATIONSHIP_FORM_VALUES), 500)` to reset after dialog close animation. The explicit `DEFAULT_RELATIONSHIP_FORM_VALUES` (with `targetRules: []`) ensures field array is properly cleared.

    5. **Polymorphic rule cache:** Create a `useRef<RelationshipFormValues['targetRules']>([])` called `cachedRulesRef`. In the submit handler or via a watched value, when `allowPolymorphic` changes:
       - CONTEXT.md decision for create mode: toggle ON caches current rules then clears; toggle OFF restores from cache
       - CONTEXT.md decision for edit mode: toggle ON clears (no restoration); toggle OFF starts fresh (empty list)
       - Expose or handle this logic via a `handlePolymorphicToggle` callback returned from the hook. Alternatively, handle in the form component (Plan 02) by watching `allowPolymorphic` and managing the cache there. For this hook, just include the `cachedRulesRef` and a helper:
       ```typescript
       const cachedRulesRef = useRef<RelationshipFormValues['targetRules']>([]);
       ```
       Return `cachedRulesRef` as part of the hook return so the UI component can use it.

    6. **Submit handler (`handleSubmit`):** Wrapped in `useCallback`. Maps form values to API request:
       - Generate `id` from existing relationship or `uuid()`
       - Map `sourceLimit + targetLimit` to `EntityRelationshipCardinality` via `calculateCardinalityFromLimits()` (convert 'ONE'/'UNLIMITED' strings to `RelationshipLimit.SINGULAR`/`RelationshipLimit.MANY` first)
       - Map `targetRules`: if `allowPolymorphic`, send `[]`. Otherwise map each rule:
         - `id: rule.id`
         - `targetEntityTypeId: rule.targetEntityTypeKey ? EntityTypeService.resolveEntityTypeId(availableTypes, rule.targetEntityTypeKey) : undefined`
         - `semanticTypeConstraint: rule.semanticTypeConstraint`
         - `cardinalityOverride: rule.cardinalityOverride`
         - `inverseVisible: rule.inverseVisible`
         - `inverseName: rule.inverseName`
       - Flatten icon: `iconType: values.icon.type, iconColour: values.icon.colour`
       - Map semantics: `semantics: values.semanticDefinition ? { definition: values.semanticDefinition, tags: [] } : undefined`
       - Construct `SaveRelationshipDefinitionRequest` with `type: EntityTypeRequestDefinition.SaveRelationship`, `key: entityType.key`
       - Wrap in `SaveTypeDefinitionRequest` with `index: undefined, definition`
       - Call `saveDefinition(request)` from `useSaveDefinitionMutation`

    7. **Mutation hook:** Use `useSaveDefinitionMutation(workspaceId, { onSuccess: () => onSave() })` -- same pattern as current code.

    **Imports to use (all from domain barrels, NOT direct model imports):**
    - `@/lib/types/entity` for: `EntityType`, `RelationshipDefinition`, `EntityRelationshipCardinality`, `SaveRelationshipDefinitionRequest`, `SaveTargetRuleRequest`, `SaveTypeDefinitionRequest`, `EntityTypeRequestDefinition`, `RelationshipLimit`, `SemanticGroup`
    - `@/lib/types/common` for: `IconType`, `IconColour`
    - `react-hook-form` for: `useForm`, `UseFormReturn`, `useFieldArray`, `UseFieldArrayReturn`
    - `@hookform/resolvers/zod` for: `zodResolver`
    - `zod` for: `z`
    - `@/lib/util/utils` for: `uuid`
    - Local imports for: `calculateCardinalityFromLimits`, `processCardinalityToLimits`, `useSaveDefinitionMutation`, `EntityTypeService`

    **Do NOT import:** `EntityRelationshipDefinition` (old type), `EntityTypeRelationshipType` (removed), `isUUID` from `validator` (no longer needed), `useRelationshipOverlapDetection`
  </action>
