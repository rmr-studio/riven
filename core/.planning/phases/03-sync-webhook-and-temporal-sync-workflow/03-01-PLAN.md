    **1. Create DTOs** in `models/integration/sync/` package:

    `IntegrationSyncWorkflowInput` data class with: connectionId (UUID), workspaceId (UUID), integrationId (UUID), nangoConnectionId (String), providerConfigKey (String), model (String), modifiedAfter (String?). These are all passed from the webhook payload and connection entity.

    `SyncProcessingResult` data class with: entityTypeId (UUID), cursor (String?), recordsSynced (Int), recordsFailed (Int), lastErrorMessage (String?), success (Boolean). Used as return type from fetchAndProcessRecords activity and input to finalizeSyncState.

    `RelationshipPending` data class with: sourceEntityId (UUID), relationshipDefinitionKey (String), targetExternalIds (List of String). Collected during Pass 1 upsert, consumed by Pass 2 relationship resolution.

    **2. Create workflow interface** `IntegrationSyncWorkflow` in `service/integration/sync/`:
    - `@WorkflowInterface` annotation
    - Single `@WorkflowMethod fun execute(input: IntegrationSyncWorkflowInput)` — void return (results written to sync state by activity)

    **3. Create activity interface** `IntegrationSyncActivities` in `service/integration/sync/`:
    - `@ActivityInterface` annotation
    - `@ActivityMethod fun transitionToSyncing(connectionId: UUID, workspaceId: UUID)` — lightweight status write
    - `@ActivityMethod fun fetchAndProcessRecords(input: IntegrationSyncWorkflowInput): SyncProcessingResult` — the long-running paginated fetch+process
    - `@ActivityMethod fun finalizeSyncState(connectionId: UUID, entityTypeId: UUID, result: SyncProcessingResult)` — single sync state write on completion

    **4. Create workflow implementation** `IntegrationSyncWorkflowImpl` in `service/integration/sync/`:
    - NOT a Spring bean — Temporal manages lifecycle. Constructor takes `RetryConfig` parameter (injected via factory)
    - Use `Workflow.getLogger()` for logging
    - In `execute()`: create activity stub via `Workflow.newActivityStub()` with:
      - `startToCloseTimeout` = 30 minutes (long-running fetch)
      - `heartbeatTimeout` = 30 seconds
      - `retryOptions` from the injected RetryConfig (maxAttempts=3, initialInterval=30s, backoff=2.0)
    - Call three activities in sequence: `transitionToSyncing`, `fetchAndProcessRecords`, `finalizeSyncState`
    - Pass `result.entityTypeId` and `result` to finalizeSyncState

    **5. Update WorkflowRetryConfigurationProperties:**
    - Add `val integrationSync: RetryConfig = RetryConfig(maxAttempts = 3, initialIntervalSeconds = 30, backoffCoefficient = 2.0, maxIntervalSeconds = 120)` field

    **6. Update application.yml:**
    - Add under `riven.workflow.retry`:
      ```yaml
      integration-sync:
        max-attempts: 3
        initial-interval-seconds: 30
        backoff-coefficient: 2.0
        max-interval-seconds: 120
      ```

    **7. Update TemporalWorkerConfiguration:**
    - Add `const val INTEGRATION_SYNC_QUEUE = "integration.sync"` to companion object
    - Add constructor parameter: `private val integrationSyncActivities: IntegrationSyncActivities` (will be the Spring bean impl from Plan 02; for now this will cause a compile warning but NOT a runtime error since the bean will exist before the app starts after Plan 02)
    - IMPORTANT: Since IntegrationSyncActivitiesImpl does not exist yet, make the `integrationSyncActivities` parameter nullable (`IntegrationSyncActivities?`) with default null, and conditionally register if not null.     - Register new worker in `workerFactory()`:
      ```kotlin

      syncWorker.registerWorkflowImplementationFactory(IntegrationSyncWorkflow::class.java) {

      integrationSyncActivities?.let { syncWorker.registerActivitiesImplementations(it) }
      ```

    **8. Replace handleSyncEvent stub in NangoWebhookService:**
    - Add `WorkflowClient` as constructor parameter (import from `io.temporal.client`)
    - Replace the stub body with:
      1. Extract `connectionId` (nango connection ID string) and `model` from payload. If either is null, log warning and return
      2. Look up internal connection via `connectionRepository.findByNangoConnectionId(nangoConnectionId)`. If null, log warning and return
      3. Build deterministic workflow ID: `"sync-${connection.id}-${model}"`
      4. Create workflow stub via `workflowClient.newWorkflowStub(IntegrationSyncWorkflow::class.java, WorkflowOptions)` with `setWorkflowId(workflowId)` and `setTaskQueue(INTEGRATION_SYNC_QUEUE)`
      5. Call `WorkflowClient.start { stub.execute(IntegrationSyncWorkflowInput(...)) }` passing connectionId, workspaceId, integrationId, nangoConnectionId, providerConfigKey, model, modifiedAfter from payload
      6. Catch `WorkflowExecutionAlreadyStarted` — log at INFO level ("already running — ignoring duplicate webhook") and return. This is expected and is the dedup mechanism.
      7. Log successful dispatch at INFO level

    **AVOID:** Do not use `@PreAuthorize` on webhook service methods — webhook security is handled by the HMAC filter. Do not catch generic Exception around the Temporal dispatch — let it propagate to the outer try-catch in `handleWebhook()`.
