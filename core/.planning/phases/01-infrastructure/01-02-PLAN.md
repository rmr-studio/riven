       -- Trigram GIN index for fuzzy attribute value matching (identity resolution)
       -- Expression MUST be (value->>'value') not (value::text) — extracts scalar string from JSONB
       -- Partial: only indexes non-deleted attributes
       CREATE INDEX IF NOT EXISTS idx_entity_attributes_trgm

       ```
       CRITICAL: The expression MUST be `(value->>'value')` — using `(value::text)` is a silent failure that indexes the JSON envelope instead of the scalar value (locked decision).

       - `updated_by` UUID

       **identity_clusters** — groups of entities confirmed as same identity:

       - `name` TEXT (optional display name)

       - `deleted_at` TIMESTAMPTZ

       - `updated_at` TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP
       - `created_by` UUID
       - `updated_by` UUID

       **identity_cluster_members** — join table linking entities to clusters:
       - `id` UUID PK DEFAULT uuid_generate_v4()

       - `entity_id` UUID NOT NULL FK -> entities(id) ON DELETE CASCADE
       - `joined_at` TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP
       - `joined_by` UUID

       -- Enforce canonical ordering: source_entity_id must be less than target_entity_id
       -- Prevents (A,B) and (B,A) as separate suggestions
       ALTER TABLE match_suggestions
           ADD CONSTRAINT chk_match_suggestions_canonical_order

       -- Unique pair constraint per workspace (only active/non-deleted suggestions)
       CREATE UNIQUE INDEX IF NOT EXISTS uq_match_suggestions_pair
           ON match_suggestions (workspace_id, source_entity_id, target_entity_id)
           WHERE deleted = false;
       ```
