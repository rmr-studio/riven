**1. NangoWebhookHmacFilter.kt** (HOOK-01, HOOK-05):

Create `src/main/kotlin/riven/core/filter/integration/NangoWebhookHmacFilter.kt`:

- Extends `OncePerRequestFilter`
- Constructor: `NangoConfigurationProperties`, `KLogger` (injected via prototype bean from LoggerConfig)
- `doFilterInternal()`:
  1. Read raw body bytes: `request.inputStream.readBytes()`
  2. Get signature header: `request.getHeader("X-Nango-Hmac-Sha256")`
  3. If header is null or blank: `response.sendError(401, "Missing webhook signature")` and return (do NOT call filterChain)
  4. Compute expected HMAC: `Mac.getInstance("HmacSHA256")` with `SecretKeySpec(nangoProperties.secretKey.toByteArray(), "HmacSHA256")`, `doFinal(bodyBytes)`, hex-encode result
  5. Compare using `MessageDigest.isEqual(expectedBytes, actualBytes)` where both are hex string bytes — use constant-time comparison
  6. On mismatch: `response.sendError(401, "Invalid webhook signature")` and return
  7. On match: wrap request in `CachedBodyHttpServletRequest(request, bodyBytes)` and call `filterChain.doFilter(wrappedRequest, response)`

- Create inner class `CachedBodyHttpServletRequest` extending `HttpServletRequestWrapper`:
  - Constructor takes `HttpServletRequest` and `ByteArray`
  - Override `getInputStream()` to return a `ServletInputStream` wrapping `ByteArrayInputStream(bodyBytes)`
  - Override `getReader()` to return `BufferedReader(InputStreamReader(getInputStream()))`

- HMAC hex encoding: `Mac.doFinal(bytes).joinToString("") { "%02x".format(it) }` — produces lowercase hex string matching Nango's format

**CRITICAL ANTI-PATTERNS TO AVOID:**
- Do NOT use `ContentCachingRequestWrapper` — it caches AFTER downstream reads, but we need bytes BEFORE. Use the custom wrapper described above.
- Do NOT call `filterChain.doFilter` after a signature failure — return immediately.
- Do NOT use `String.equals()` for signature comparison — use `MessageDigest.isEqual()`.
- Do NOT use `nangoProperties.webhookSecret` — use `nangoProperties.secretKey` per user decision.

Public method `handleWebhook(payload: NangoWebhookPayload)`:

```kotlin
fun handleWebhook(payload: NangoWebhookPayload) {
    when (payload.type) {
        "auth" -> handleAuthEvent(payload)
        "sync" -> handleSyncEvent(payload)
        else -> logger.info { "Ignoring unknown Nango webhook type: ${payload.type}" }
    }
}
```

Private method `handleAuthEvent(payload: NangoWebhookPayload)`:
1. Extract tags: `val tags = payload.tags`. If tags is null, log error and return.
2. Parse UUIDs from tags:
   - `val userId = tags.endUserId?.let { UUID.fromString(it) }` — if null, log error and return (HOOK-06)
   - `val workspaceId = tags.organizationId?.let { UUID.fromString(it) }` — if null, log error and return (HOOK-06)
   - `val integrationDefinitionId = tags.endUserEmail?.let { UUID.fromString(it) }` — if null, log error and return (HOOK-06)
   - Wrap each `UUID.fromString()` in try-catch for `IllegalArgumentException` — invalid UUID format logs error and returns
3. Get `nangoConnectionId = payload.connectionId` — if null, log error and return
4. Check `payload.success` — if false or null, log warning and return (auth failed at Nango side)
5. Call `handleAuthWebhookTransaction(workspaceId, integrationDefinitionId, userId, nangoConnectionId)`

Private method `handleAuthWebhookTransaction(workspaceId, integrationDefinitionId, userId, nangoConnectionId)` — annotated with `@Transactional`:
1. Validate integration definition exists: `val definition = definitionRepository.findById(integrationDefinitionId).orElse(null)` — if null, log error and return (HOOK-06, graceful)
2. Create or reconnect connection:
   - `val existing = connectionRepository.findByWorkspaceIdAndIntegrationId(workspaceId, integrationDefinitionId)`
   - If existing is null: create new `IntegrationConnectionEntity(workspaceId, integrationId=integrationDefinitionId, nangoConnectionId, status=CONNECTED)` and save
   - If existing status is DISCONNECTED or FAILED: set status to CONNECTED, update nangoConnectionId, save
   - If existing status is CONNECTED: update nangoConnectionId if different, return existing (idempotent)
   - Otherwise: log warning about unexpected status, still update nangoConnectionId and save (don't reject — webhook must succeed)
3. Find or create installation:
   - `val installation = installationRepository.findByWorkspaceIdAndIntegrationDefinitionId(workspaceId, integrationDefinitionId)`
   - If null, check for soft-deleted: `installationRepository.findSoftDeletedByWorkspaceIdAndIntegrationDefinitionId(workspaceId, integrationDefinitionId)`
   - If soft-deleted exists: restore it (set deleted=false, deletedAt=null)
   - If neither exists: create new `WorkspaceIntegrationInstallationEntity(workspaceId, integrationDefinitionId, manifestKey=definition.slug, installedBy=userId)`
   - Set status to `InstallationStatus.ACTIVE` and save
4. Trigger materialization (with try-catch per CONTEXT.md decision):
   ```kotlin
   try {
       templateMaterializationService.materializeIntegrationTemplates(workspaceId, definition.slug, integrationDefinitionId)
   } catch (e: Exception) {
       logger.error(e) { "Materialization failed for workspace=$workspaceId, integration=${definition.slug}" }
       installation.status = InstallationStatus.FAILED
       installationRepository.save(installation)
       // Do NOT rethrow — transaction commits with FAILED status, connection preserved
   }
   ```
5. Log activity for the connection creation/update.

**CRITICAL:** The single `@Transactional` method works because the catch block does a compensation write (FAILED) and does NOT rethrow. Spring only rolls back on uncaught exceptions. The connection and FAILED installation commit together.

Private method `handleSyncEvent(payload: NangoWebhookPayload)`:
- Phase 3 stub: `logger.info { "Received sync webhook event for model=${payload.model}, syncName=${payload.syncName} — dispatch not yet implemented" }`
- Return without action.

**IMPORTANT ANTI-PATTERNS:**
- Do NOT call `authTokenService.getUserId()` — there is no JWT in the webhook path. userId comes from tags.
- Do NOT use `@PreAuthorize` on any method — no security context in webhook path.
- Do NOT rethrow in the materialization catch block — this would roll back the transaction and lose the connection.
- Do NOT throw exceptions from handleAuthEvent or handleSyncEvent — the controller must always return 200.
