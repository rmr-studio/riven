Takes `tokens: List<String>`, filters to tokens with length >= 2, issues `SELECT dmetaphone(:token)` via `entityManager.createNativeQuery` for each token, collects non-empty results into a `Set<String>`. Follow the pattern in 05-RESEARCH.md Pattern 1.

**3. Add `findPhoneticCandidates` private method:**
- Tokenize normalizedValue via `lowercase().split(Regex("\\s+"))`, filter non-empty
- Call `computePhoneticCodes(tokens)` — early return `emptyList()` if empty set (prevents empty-collection SQL error)
- Native SQL query using EXISTS with `regexp_split_to_table(LOWER(ea.value->>'value'), '\\s+')` and `dmetaphone(token) = ANY(:phoneticCodes)` with guards: `LENGTH(token) >= 2` and `dmetaphone(token) != ''`
- Fixed similarity score of 0.85
- Returns `List<CandidateMatch>` with `matchSource = MatchSource.PHONETIC`
- Uses `LIMIT $CANDIDATE_LIMIT` — no DISTINCT ON (per CAND-02 pattern, let mergeCandidates dedup)
- Pass `phoneticCodes.toTypedArray()` to `setParameter`. If Hibernate type inference fails, use Hypersistence `StringArrayType.INSTANCE` as fallback (see 05-RESEARCH.md ANY() binding section).
- Add KDoc matching the style of existing private query methods (findNicknameCandidates as reference).

  Returns combined list.

`findEmailCandidates(triggerEntityId, workspaceId, normalizedValue, signalType)`:

`findDefaultCandidates(triggerEntityId, workspaceId, normalizedValue, signalType)`:

**5. Replace inline if-chains in `findCandidates()` loop with when() dispatch:**

    MatchSignalType.EMAIL             -> findEmailCandidates(triggerEntityId, workspaceId, normalizedValue, signalType)

    MatchSignalType.CUSTOM_IDENTIFIER -> findDefaultCandidates(triggerEntityId, workspaceId, normalizedValue, signalType)
