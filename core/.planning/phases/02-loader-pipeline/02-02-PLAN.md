  <action>
    **Create ManifestResolverService** in `service.catalog` package. Constructor-inject `ObjectMapper` and `KLogger`. No repository dependencies -- this is pure in-memory transformation.

    Public method:
    - `fun resolveManifest(scanned: ScannedManifest, modelIndex: Map<String, JsonNode>): ResolvedManifest` -- orchestrates resolution for a single manifest. For MODEL type, parse the single entity type from the JSON directly. For TEMPLATE type, iterate `entityTypes` array, resolve each entry (inline or $ref). For INTEGRATION type, parse inline entity types. Then normalize relationships, validate relationships, validate field mappings. If any $ref is unresolvable or any relationship key is invalid, return a ResolvedManifest with `stale = true` and empty children.

    Private methods organized by concern:

    **// ------ Entity Type Resolution ------**
    - `resolveEntityTypes(json: JsonNode, type: ManifestType, modelIndex: Map<String, JsonNode>): Pair<List<ResolvedEntityType>, Boolean>` -- returns resolved list + whether any $ref failed. For MODEL: parse the root JSON as a single entity type. For TEMPLATE: iterate `entityTypes`, each entry either has `$ref` (resolve from modelIndex, optionally apply extend) or is inline. For INTEGRATION: iterate `entityTypes` as inline.
    - `resolveRefEntityType(entry: JsonNode, modelIndex: Map<String, JsonNode>): ResolvedEntityType?` -- looks up `$ref` value (strip `models/` prefix) in modelIndex. Returns null if missing.
    - `applyExtend(base: JsonNode, extend: JsonNode): JsonNode` -- shallow additive merge. Scalar overrides: `description`, `icon`, `semanticGroup`, `identifierKey`. Attributes: add new keys from extend, skip keys already in base. Semantic tags: append extend tags to base tags.
    - `parseEntityType(json: JsonNode, readonlyDefault: Boolean = false): ResolvedEntityType` -- extracts all fields from a JsonNode into a ResolvedEntityType. Use `readonlyDefault = true` for integration entity types.

    **// ------ Relationship Resolution ------**
    - `normalizeRelationships(json: JsonNode, manifestType: ManifestType): List<NormalizedRelationship>?` -- iterates `relationships` array. For each, calls `normalizeRelationship`. Returns null if any relationship has mutual exclusivity violation (both shorthand and full format).
    - `normalizeRelationship(rel: JsonNode, manifestType: ManifestType): NormalizedRelationship?` -- checks mutual exclusivity. If shorthand: convert `targetEntityTypeKey` + `cardinality` to `targetRules[{targetEntityTypeKey}]` + `cardinalityDefault`. If full: parse `targetRules` directly. Apply protected default: `true` for INTEGRATION, `false` for TEMPLATE. Explicit `protected` field overrides default.
    - `validateRelationships(relationships: List<NormalizedRelationship>, entityTypeKeys: Set<String>): Boolean` -- checks all sourceEntityTypeKey and targetRules[].targetEntityTypeKey exist in entityTypeKeys set. Checks for duplicate relationship keys. Returns false if any validation fails.

    **// ------ Field Mapping Resolution ------**
    - `resolveFieldMappings(json: JsonNode, entityTypeKeys: Map<String, Set<String>>): List<ResolvedFieldMapping>` -- for INTEGRATION only. Iterates `fieldMappings` array. For each entry, validate that mapping keys (the attribute keys) exist in the target entity type's attribute set. Skip invalid mapping keys with WARN. The `entityTypeKeys` parameter maps entity type key to its set of attribute keys.

    **TEST-04 (relationship validation):** 3 tests -- valid keys pass, missing source key returns stale, duplicate relationship keys detected.

    **LOAD-07 (protected defaults):** 2 tests -- INTEGRATION defaults protected=true, TEMPLATE defaults protected=false.

    **Field mapping validation:** 2 tests -- valid mapping key included, invalid mapping key skipped.

    Total: ~17 test cases. Use helper methods to build JsonNode fixtures to keep tests readable.
