  <behavior>
    - New manifest (key+type not in DB): creates ManifestCatalogEntity + all children, stale=false, lastLoadedAt=now
    - Existing manifest (key+type found): updates ManifestCatalogEntity fields, deletes all children, reinserts fresh children, stale=false, lastLoadedAt=now
    - Stale manifest (ResolvedManifest.stale=true): upserts catalog row with stale=true, does NOT insert children
    - Child deletion order: target_rules (by relationship IDs) -> relationships (by manifestId), semantic_metadata (by entity type IDs) -> entity_types (by manifestId), field_mappings (by manifestId)
    - Entity types persisted with all fields mapped from ResolvedEntityType
    - Relationships persisted with target rules as separate rows
    - Semantic metadata persisted for entity types that have semantics
    - Field mappings persisted with raw mappings map as JSONB
    - Entire method is @Transactional -- one manifest per transaction
  </behavior>
  <action>
    **Create ManifestUpsertService** in `service.catalog` package. Constructor-inject all 6 catalog repositories and `KLogger`. No scanner/resolver dependencies -- this service only receives `ResolvedManifest`.

    Main public method:
    ```kotlin
    @Transactional
    fun upsertManifest(resolved: ResolvedManifest)
    ```

    Implementation flow:
    1. Find or create catalog entry: `manifestCatalogRepository.findByKeyAndManifestType(resolved.key, resolved.type)`. If found, update mutable fields (`name`, `description`, `manifestVersion`, `lastLoadedAt`, `stale`). If not found, create new `ManifestCatalogEntity`. Save and flush to get ID.
    2. If `resolved.stale` is true, stop here -- do not persist children.
    3. Delete existing children in cascading order to avoid FK violations:
       - Find existing relationship IDs -> delete target rules by those IDs
       - Delete relationships by manifestId
       - Find existing entity type IDs -> delete semantic metadata by those IDs
       - Delete entity types by manifestId
       - Delete field mappings by manifestId
    4. Insert fresh entity types from `resolved.entityTypes` -> map each `ResolvedEntityType` to `CatalogEntityTypeEntity` with `manifestId`. Save all. For each entity type with `semantics`, create `CatalogSemanticMetadataEntity` rows.
    5. Insert fresh relationships from `resolved.relationships` -> map each `NormalizedRelationship` to `CatalogRelationshipEntity`. Save each, then create `CatalogRelationshipTargetRuleEntity` rows for each `targetRule`.
    6. Insert fresh field mappings from `resolved.fieldMappings` -> map each `ResolvedFieldMapping` to `CatalogFieldMappingEntity` with `manifestId`.

    Use enums for iconType/iconColour/semanticGroup/cardinality: parse string values from ResolvedEntityType/NormalizedRelationship using `valueOf()`. Use `IconType.valueOf(resolved.iconType)`, `IconColour.valueOf(resolved.iconColour)`, `SemanticGroup.valueOf(resolved.semanticGroup)`.

    Extract private helper methods to keep the public method readable:
    - `findOrCreateCatalog(resolved: ResolvedManifest): ManifestCatalogEntity`
    - `deleteChildren(manifestId: UUID)`
    - `insertEntityTypes(manifestId: UUID, entityTypes: List<ResolvedEntityType>)`
    - `insertRelationships(manifestId: UUID, relationships: List<NormalizedRelationship>)`
    - `insertFieldMappings(manifestId: UUID, fieldMappings: List<ResolvedFieldMapping>)`

    **Create ManifestUpsertServiceTest** -- mock all 6 repositories. Test:
    - `upsertManifest creates new catalog entry and children` -- verify save called on catalog repo, entity type repo, relationship repo, etc.
    - `upsertManifest updates existing catalog entry and reinserts children` -- mock findByKeyAndManifestType to return existing entity, verify delete methods called before inserts
    - `upsertManifest with stale manifest only upserts catalog row` -- verify no child insert calls when stale=true
    - `upsertManifest deletes children in correct order` -- verify delete order via InOrder verification (target rules before relationships, semantic metadata before entity types)
  </action>

    `fun loadAllManifests()` -- the core orchestration. NO `@Transactional` on this method (per CONTEXT/RESEARCH anti-pattern). Steps:

    **// ------ Mark All Stale ------**
    Call `manifestCatalogRepository.markAllStale()`. Must be wrapped in its own `@Transactional` context -- since the method is `@Modifying`, it needs a transaction. Use a separate private `@Transactional` helper or call a transactional repository method.

    IMPORTANT: `markAllStale()` is a `@Modifying` JPQL query that requires an active transaction. Since `loadAllManifests()` is NOT `@Transactional`, the `@Modifying` query in the repository will fail without a transaction. Solution: either make `markAllStale()` work via `@Transactional` on the repository method itself (Spring Data repos support this), OR create a small `@Transactional` wrapper method in this service that calls `markAllStale()`. The simplest approach: add `@Transactional` annotation directly on `markAllStale()` in `ManifestCatalogRepository`.

    **// ------ Load Models ------**
    ```kotlin
    val scannedModels = scannerService.scanModels()
    val modelIndex = mutableMapOf<String, JsonNode>()
    var modelsLoaded = 0; var skipped = 0
    for (scanned in scannedModels) {
        try {
            val resolved = resolverService.resolveManifest(scanned, emptyMap())
            upsertService.upsertManifest(resolved)
            if (!resolved.stale) {
                modelIndex[scanned.key] = scanned.json
                modelsLoaded++
            }
        } catch (e: Exception) {
            logger.warn(e) { "Failed to load model manifest: ${scanned.key}" }
            skipped++
        }
    }
    ```

    **// ------ Load Templates ------**
    Same pattern but pass `modelIndex` to `resolverService.resolveManifest()`.

    **// ------ Load Integrations ------**
    Same pattern with empty modelIndex (integrations don't use $ref).

    **// ------ Sync Integration Definitions ------**
    After all manifests loaded:
    ```kotlin
    syncIntegrationDefinitionsStale()
    ```
    Private method that queries `manifestCatalogRepository.findByManifestType(ManifestType.INTEGRATION)`, then for each, finds matching `IntegrationDefinitionEntity` by slug and syncs its `stale` field. Use `integrationDefinitionRepository.findBySlug(catalogEntry.key)`. If found, set `entity.stale = catalogEntry.stale` and save. If not found, skip (integration definition may not exist yet).

    This sync method needs its own `@Transactional` boundary. Add `@Transactional` on the private method.

    **// ------ Summary Log ------**
    ```kotlin
    val staleCount = manifestCatalogRepository.findByStaleTrue().size
    logger.info { "Manifest load complete: $modelsLoaded models, $templatesLoaded templates, $integrationsLoaded integrations loaded. $staleCount stale. $skipped skipped." }
    ```

    - `loadAllManifests logs summary with correct counts` -- verify logger.info called with expected message pattern
