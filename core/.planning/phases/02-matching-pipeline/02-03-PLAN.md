    **IdentityMatchSuggestionService.kt** (`riven.core.service.identity`):
    - Constructor inject: `MatchSuggestionRepository`, `ActivityService`, `KLogger`
    - `@Transactional` on mutation methods.
    - No `@PreAuthorize` — called from Temporal activity, workspace ID passed explicitly. The rejectSuggestion method will get `@PreAuthorize` added in Phase 5 when the REST API wires it.

    **Public methods:**

    `fun persistSuggestions(workspaceId: UUID, scoredCandidates: List<ScoredCandidate>, userId: UUID?): Int`
    - For each ScoredCandidate, call `createOrResuggest()`. Count successes. Return count.
    - `userId` is nullable — Temporal activities may not have JWT context. Pass to activity logging (use null for system-generated).

    `fun rejectSuggestion(suggestionId: UUID, userId: UUID): MatchSuggestion`
    - This is the reject write path required by SUGG-03. Without it, `rejectionSignals` is never populated and re-suggestion (SUGG-04) cannot work.
    - Implementation:
      1. Fetch suggestion via `ServiceUtil.findOrThrow { repository.findById(suggestionId) }`.
      2. Validate current status is PENDING — if already REJECTED or CONFIRMED, throw `ConflictException("Suggestion is already ${entity.status}")`.
      3. Copy the current `signals` to `rejectionSignals` as a snapshot: `entity.rejectionSignals = mapOf("signals" to entity.signals, "confidenceScore" to entity.confidenceScore.toDouble())`. This stores the signal state at rejection time so re-suggestion can compare against it.
      4. Set `entity.status = MatchSuggestionStatus.REJECTED`.
      5. Set `entity.resolvedBy = userId` and `entity.resolvedAt = ZonedDateTime.now()`.
      6. Save: `repository.save(entity)`.
      7. Log activity: `activityService.logActivity(Activity.MATCH_SUGGESTION, OperationType.UPDATE, userId, entity.workspaceId, ApplicationEntityType.MATCH_SUGGESTION, requireNotNull(entity.id) { "Suggestion ID must not be null" }, details)` with details map containing "action" to "rejected", "sourceEntityId", "targetEntityId", "confidenceScore".
      8. Return `entity.toModel()`.

    **Private methods:**

    `private fun createOrResuggest(candidate: ScoredCandidate, workspaceId: UUID, userId: UUID?): MatchSuggestion?`
    Implementation per RESEARCH.md Pattern 5:
    1. Apply canonical UUID ordering: `val (sourceId, targetId) = canonicalOrder(candidate.sourceEntityId, candidate.targetEntityId)`
    2. Check for active suggestion: `repository.findActiveSuggestion(workspaceId, sourceId, targetId)` — if non-null, return null (idempotent skip, log debug).
    3. Check for rejected suggestion: `repository.findRejectedSuggestion(workspaceId, sourceId, targetId)` — if non-null and `candidate.compositeScore <= rejected.confidenceScore.toDouble()`, return null (not stronger, log debug).
    4. If rejected exists and score is higher: soft-delete the rejected row (`rejected.deleted = true; rejected.deletedAt = ZonedDateTime.now(); repository.save(rejected)`).
    5. Build `MatchSuggestionEntity` with: workspaceId, sourceId, targetId, PENDING status, `BigDecimal.valueOf(candidate.compositeScore)` as confidenceScore, `candidate.signals.map { it.toMap() }` as signals.
    6. Call `createIfNotExists(entity)`.
    7. If suggestion created, log activity.

    `private fun createIfNotExists(entity: MatchSuggestionEntity): MatchSuggestion?`
    Per RESEARCH.md Pattern 4:
    - `try { repository.saveAndFlush(entity).toModel() } catch (e: DataIntegrityViolationException) { logger.info { "Duplicate suggestion skipped" }; null }`

    `private fun canonicalOrder(a: UUID, b: UUID): Pair<UUID, UUID> = if (a < b) a to b else b to a`

    `private fun logSuggestionCreated(suggestion: MatchSuggestion, userId: UUID?)`
    - Call `activityService.logActivity(Activity.MATCH_SUGGESTION, OperationType.CREATE, userId, suggestion.workspaceId, ApplicationEntityType.MATCH_SUGGESTION, suggestion.id, details)`.
    - Details map: "sourceEntityId", "targetEntityId", "confidenceScore", "signalCount".
