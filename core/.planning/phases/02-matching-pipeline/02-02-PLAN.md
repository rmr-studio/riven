    **IdentityMatchCandidateService.kt** (`riven.core.service.identity`):
    - Constructor inject: `EntityManager` (for native queries), `KLogger`
    - No `@PreAuthorize` needed — this is called from Temporal activity, not a controller. Workspace ID is passed explicitly.

    Public method 1: `fun findCandidates(triggerEntityId: UUID, workspaceId: UUID): List<CandidateMatch>`

    Implementation:
    1. First, query the trigger entity's IDENTIFIER-classified attributes via the shared private method `queryTriggerIdentifierAttributes(triggerEntityId, workspaceId)` (see below).
    2. For each IDENTIFIER attribute, run the two-phase pg_trgm candidate query (per RESEARCH.md Pattern 2):
       ```sql
       SELECT DISTINCT ON (ea.entity_id)
           ea.entity_id AS candidate_entity_id,
           ea.attribute_id AS candidate_attribute_id,
           ea.value->>'value' AS candidate_value,
           similarity(ea.value->>'value', :inputValue) AS sim_score
       FROM entity_attributes ea
       JOIN entity_type_semantic_metadata sm
           ON sm.workspace_id = :workspaceId
          AND sm.target_type = 'ATTRIBUTE'
          AND sm.target_id = ea.attribute_id
          AND sm.classification = 'IDENTIFIER'
          AND sm.deleted = false
       WHERE ea.workspace_id = :workspaceId
         AND ea.entity_id != :triggerEntityId
         AND ea.deleted = false
         AND (ea.value->>'value') % :inputValue
         AND similarity(ea.value->>'value', :inputValue) > :threshold
       ORDER BY ea.entity_id, sim_score DESC
       LIMIT 50
       ```
    3. Normalize input values before querying: trim whitespace, lowercase.
    4. Map each result row to a `CandidateMatch`, deriving `signalType` from the trigger attribute's SchemaType via `MatchSignalType.fromSchemaType()`.
    5. Merge results across all trigger attributes: group by (candidateEntityId, signalType), keep the match with highest similarityScore per group.
    6. Return the merged list.

    Public method 2: `fun getTriggerAttributes(entityId: UUID, workspaceId: UUID): Map<MatchSignalType, String>`

    Implementation:
    1. Call the shared private method `queryTriggerIdentifierAttributes(entityId, workspaceId)` to get the trigger entity's IDENTIFIER-classified attributes.
    2. For each result row, derive `MatchSignalType` from SchemaType via `MatchSignalType.fromSchemaType()`, and extract the normalized attribute value (trim + lowercase).
    3. If multiple attributes map to the same MatchSignalType, keep the first (or any — the scoring service handles best-match logic).
    4. Return the map.

    Private shared method: `private fun queryTriggerIdentifierAttributes(entityId: UUID, workspaceId: UUID): List<Array<Any>>`

    This extracts the common SQL query used by both `findCandidates` and `getTriggerAttributes`:
    ```sql
    SELECT ea.attribute_id, ea.value->>'value' AS attr_value, ea.schema_type
    FROM entity_attributes ea
    JOIN entity_type_semantic_metadata sm
        ON sm.workspace_id = :workspaceId
       AND sm.target_type = 'ATTRIBUTE'
       AND sm.target_id = ea.attribute_id
       AND sm.classification = 'IDENTIFIER'
       AND sm.deleted = false
    WHERE ea.entity_id = :entityId
      AND ea.workspace_id = :workspaceId
      AND ea.deleted = false
    ```

    Use `EntityManager.createNativeQuery()` with `.setParameter()` for parameterized queries. Use `pg_trgm.similarity_threshold` set to 0.3 (the default) — the `%` operator uses whatever the session threshold is, which defaults to 0.3.

    **IdentityMatchScoringService.kt** (`riven.core.service.identity`):
    - Constructor inject: `KLogger`
    - No Spring security annotations — pure computation, no DB access.
    - Companion object: `const val MINIMUM_SCORE_THRESHOLD = 0.5`

    Public method: `fun scoreCandidates(triggerEntityId: UUID, triggerAttributes: Map<MatchSignalType, String>, candidates: List<CandidateMatch>): List<ScoredCandidate>`

    Parameters:
    - `triggerEntityId`: the entity being matched
    - `triggerAttributes`: map of signal type to normalized attribute value for the trigger entity (e.g., EMAIL -> "john@example.com")
    - `candidates`: raw candidate matches from IdentityMatchCandidateService

    Implementation:
    1. Group candidates by `candidateEntityId`.
    2. For each candidate entity:
       a. For each signal type present in candidates for this entity, find the best match (highest similarityScore) — this handles "best match per type" requirement.
       b. Build a `MatchSignal` for each matched type: type, sourceValue (from triggerAttributes), targetValue (from CandidateMatch.candidateValue), similarity (from CandidateMatch.similarityScore), weight (from MatchSignalType.DEFAULT_WEIGHTS).
       c. Filter out signals where similarity == 0.0.
       d. Compute composite score: `Sum(signal.similarity * signal.weight) / Sum(signal.weight)` — per locked decision.
       e. If composite score >= MINIMUM_SCORE_THRESHOLD, create `ScoredCandidate(triggerEntityId, candidateEntityId, compositeScore, signals)`.
    3. Return list of scored candidates (only those above threshold).
